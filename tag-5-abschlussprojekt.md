## 📖 Option A: Das ultimative Minecraft-Handbuch

### Ziel:
Ein 10-seitiges Handbuch zu einem fortgeschrittenen Minecraft-Thema

### Struktur:
1. **Einleitung** (KI-generiert, von euch überarbeitet)
2. **Grundlagen** (mit eigenen Screenshots/Zeichnungen)
3. **Schritt-für-Schritt Anleitung** (KI-Unterstützung)
4. **Fortgeschrittene Tipps** (eigene Recherche + KI)
5. **Häufige Fehler** (aus eigenen Erfahrungen)
6. **Bonus: Kreative Variationen** (KI-Brainstorming)

### Beispiel-Themen:
- 🏰 Eine automatische Burg mit Redstone
- 🚂 Ein komplexes Minenbahnsystem
- 🌾 Eine vollautomatische Farm für jedes Item
- ⚔️ Ein Minecraft-Abenteuer-Map

### Arbeitsablauf:

**Phase 1: Planung (30 Min)**
```
ROLLE: Du bist Minecraft-Experte und Autor.

AUFGABE: Hilf mir, ein Handbuch zu planen über:
[THEMA EINFÜGEN]

FORMAT: Gliederung mit:
- Kapitelüberschriften
- Zwischenüberschriften
- Wichtige Screenshots/Zeichnungen die nötig sind

Beschränkung: Maximal 10 Seiten, kindgerecht
```

**Phase 2: Inhalte erstellen (60 Min)**
Jedes Kapitel mit KI-Hilfe schreiben, aber:
- Eigene Screenshots einfügen
- Eigene Erfahrungen ergänzen
- KI-Text überarbeiten (eigene Worte!)

**Phase 3: Design (30 Min)**
- Canva, Word oder PowerPoint nutzen
- Einheitliches Design
- Minecraft-ähnliche Farben

## 🎮 Option B: Das eigene Roblox-Spiel-Konzept

### Ziel:
Ein komplettes Game Design Document (GDD) für ein Roblox-Spiel

### Struktur:
1. **Elevator Pitch** (In einem Satz: Was ist das Spiel?)
2. **Core Loop** (Was machen Spieler immer wieder?)
3. **Features** (Wichtigste 5 Features)
4. **Visual Style** (Wie sieht es aus?)
5. **UI/UX** (Wie bedient man es?)
6. **Technische Planung** (Wie wird es gebaut?)
7. **Marketing** (Wie werdet ihr berühmt? 😄)

### Arbeitsablauf:

**Phase 1: Die Idee (20 Min)**
```
ROLLE: Du bist Game Designer bei Roblox Corporation.

AUFGABE: Wir brainstomen Spielideen für ein Spiel 
für 12-jährige, das folgendes hat:
- Teamwork-Elemente
- Kreativität
- Wiederbespielwert
- Kein Kampf/Violence

Gib mir 10 kreative Ideen mit kurzer Beschreibung.
```

**Phase 2: Auswahl & Vertiefung (30 Min)**
Eine Idee wählen und ausarbeiten:
```
Wir haben uns für diese Idee entschieden: [IDEE]

Erstelle ein detailliertes Game Design Document mit:
[Struktur von oben]

Besonders wichtig: Die drei Spieler (Maxim, Alia, Amir) 
sollen alle unterschiedliche Rollen haben, die sich 
ergänzen!
```

**Phase 3: Prototyp-Planung (40 Min)**
```
Für unser Spiel [NAME], welches [KURZBESCHREIBUNG] macht:

Was ist die ERSTE Version, die wir bauen könnten?
(Keine komplexen Features, nur das Kern-Erlebnis)

Liste auf:
1. Was muss auf jeden Fall rein?
2. Was kann später kommen?
3. Was ist das erste Script, das wir schreiben sollten?
4. Welche 3 Roblox-Features müssen wir lernen?
```

## 🧠 Option C: Das KI-gestützte Lern-Tool

### Ziel:
Ein persönliches "KI-System" für ein Lernziel

### Ideen:
- Ein Mathe-Quiz-Generator
- Ein Vokabel-Trainer
- Ein Geschichts-Zeitstrahl-Ersteller
- Ein Science-Experiment-Planer

### Arbeitsablauf:

**Phase 1: Problem identifizieren (20 Min)**
```
Ich habe Schwierigkeiten mit: [FACH/THEMA]

Meine spezifischen Probleme sind:
1. ___________________
2. ___________________

Wie könnte eine KI mir helfen, das besser zu lernen?
Gib mir 5 konkrete Ideen.
```

**Phase 2: Das "Prompt-System" erstellen (40 Min)**
Ein Master-Prompt entwickeln, den ihr immer wieder nutzen könnt:

```
ROLLE: Du bist mein persönlicher [FACH]-Tutor.

MEIN NIVEAU: Anfänger/Fortgeschritten/Experte
MEIN LERNSTIL: Ich lerne am besten durch...
  [Beispiele/Übungen/Erklärungen/Zeichnungen]

AUFGABE: Hilf mir, [KONKRETES THEMEN] zu verstehen.

REGELN:
1. Erkläre mir Konzepte mit Minecraft/Roblox-Beispielen
2. Gib mir nach jeder Erklärung eine Mini-Übung
3. Wenn ich falsch antworte, erkläre es anders
4. Feiere mit mir, wenn ich richtig antworte!

BEGINNEN: [EURE ERSTE FRAGE]
```

**Phase 3: Testen & Verbessern (30 Min)**
Das System testen und notieren:
- Was funktioniert gut?
- Was könnte besser sein?
- Welche Prompt-Änderungen helfen?

## 🎪 Präsentations-Training

### Jeder präsentiert sein Projekt (5 Min)

**Struktur:**
1. **Hook** (10 Sek): Eine spannende Einleitung
2. **Problem** (30 Sek): Was wolltet ihr lösen?
3. **Lösung** (2 Min): Was habt ihr gemacht?
4. **KI-Nutzung** (1 Min): Wie habt ihr KI eingesetzt?
5. **Learnings** (1 Min): Was habt ihr gelernt?

### Feedback-Regeln:
- 👍 Was war besonders gut?
- 💡 Was würde ich anders machen?
- ❓ Eine Frage zum Verständnis
